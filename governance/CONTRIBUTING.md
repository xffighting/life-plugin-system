# 欢迎一起把"人生外挂"做大

谢谢你愿意贡献。这个项目相信一件事：**一个人能搭一套系统，一群人能搭一个生态。**
无论你是补一个模板、改一个错字、还是翻译成你的母语，都是真实的帮助。

## 你可以贡献什么

| 类型 | 放哪 | 例子 |
|---|---|---|
| 模板 | `templates/` | 一个更好用的"周复盘"模板 |
| 提示词 | `prompts/` | 一个"季度目标拆解"提示词 |
| 自动化 | `automations/` | 一个低风险的脱敏/整理脚本 |
| 示例数据 | `obsidian/sample-vault/` | 完全虚构的示例内容 |
| 文档/翻译 | `README` / 未来的 `i18n/` | 更清晰的说明、多语种版本 |

## 怎么提（30 秒看懂）

1. Fork 本仓库。
2. 新建分支，加你的文件。
3. 提 Pull Request，标题写清"加了什么 / 解决什么"。
4. 不确定？先开一个 Issue 聊想法，再动手也行。

> 第一次贡献？找带 `good first issue` 标签的任务最适合上手。

## 贡献内容的硬要求（重要）

1. **必须通用**：能被陌生人直接用，不依赖你的私人上下文。
2. **必须脱敏**：不含任何真实姓名、公司、客户、联系方式、报价、收入、健康、家庭信息。
3. **示例必须虚构**：用 `示例公司A` / `张示例` / `example.com` 之类的占位。
4. **发前自查**：改了脚本或带数据的文件，先跑 `python3 automations/deid_gate.py --gate 你的文件`。
5. **保持类型边界**：别把实践活动（如瑜伽、钢琴等练习）当成 AI Agent 技能；别把渠道当系统目的；别把信息源当知识资产。

## Issue 与 PR 规则

- 一个 Issue 只讲一件事；带复现/示例更易处理。
- 小而专注的 PR 比大杂烩更快合并；**按质量合并，不按是否赞助**。
- 这是业余维护的项目，请给点耐心 🙏。

## 我们不接受的贡献

- 含真实隐私 / 未脱敏的内容。
- 夹带广告、推广链接、商业私货。
- 破坏类型边界、违反隐私分级的改动。
- 大段未经讨论的架构重写。

## 许可证与署名

- 本项目使用 **MIT**。提交贡献即表示你同意你的贡献以 MIT 授权（inbound = outbound）。
- 你保留署名；我们会在版本说明里致谢贡献者。

违反者，维护者有权移除其内容或参与资格。
