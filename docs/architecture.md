# 架构 · 五层 + 一脊

> 宪法定义"为什么",架构定义"怎么放、怎么连、怎么流"。
> 这是整套系统最值得一读的部分——它是让这套系统"用十年也不乱"的工程内核。

---

## 一、总览

```
                    ┌──────────────────────────────┐
                    │   内核 · 一部可修订的宪法      │
                    │   人生最高判据，冲突在此裁决    │
                    └──────────────────────────────┘
                                  │
   ┌──────────┬───────────┬───────────────┬───────────┬───────────┐
   ▼          ▼           ▼               ▼           ▼
┌────────┐ ┌────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐
│L1 输入  │→│L2 处理  │→│L3 沉淀    │→│L4 产出    │→│L5 反馈    │
│事实/想法│ │目标     │ │知识/能力  │ │项目/作品  │ │复盘       │
└────────┘ └────────┘ └──────────┘ └──────────┘ └──────────┘
                                  │
        ── 脊柱：八经纬域 · 概念字典 · 数据账册 · AI 协同 ──
```

**核心隐喻**:内核是地基,五层是楼层,脊柱穿过所有楼层,让结构稳固。

## 二、五层数据流

| 层 | 职责 | 主存内容 |
|---|---|---|
| L1 输入 | 捕获原始事实与想法 | 每日事实流、想法、互动记录 |
| L2 处理 | 用目标牵引方向 | 目标卡片(含杠杆/复利字段) |
| L3 沉淀 | 把经验变成可复用资产 | 知识资产、能力、SOP、技能 |
| L4 产出 | 用项目推进 | 项目、作品 |
| L5 反馈 | 用复盘校准 | 日省/周省/月省/岁省 |

信息沿这条流水线单向流动:进来的不会乱跑,最终都变成行动,再被复盘校准回目标。

## 三、三权分立

- **目标牵引**:目标是方向盘。没有目标,就没有项目;没有项目,就没有日常行动。
- **项目推进**:项目是发动机。每个项目必须"服务"一个目标,无目标的项目限时归档。
- **复盘校准**:复盘是校准器。每次复盘必须产出三样——事实清单、判断更新、下一步。

三者不可互相替代:目标不替代项目,项目不替代日常事实,复盘不替代下一步行动。**混用即"违宪"。**

## 四、入网七步(给人也给 Agent 的协议)

任何一份新内容,必须走完这七步才算"入网":

1. **暂存** → 扔进待归位区。
2. **判型** → 它是 事实 / 知识 / 人物 / 实践 / 其他?
3. **命名** → 按命名规范生成 `<前缀>-<语义化短名>`。
4. **建卡** → 选模板,填齐 `parent` 与 `related`。
5. **连接** → 至少 1 条上级边 + ≥1 条相关边。
6. **登记** → 同步进数据账册的注册表。
7. **归位** → 从待归位区移入正式目录。

这套流程是**给 AI Agent 执行的状态机**,而不只是给人看的规范。

## 五、双轨制(Markdown ↔ CSV)

```
Markdown 卡片 (人读 + 双链)          CSV 注册表 (机器读 + 索引)
        │                                  │
        └──────── 入网时同步 ──────────────┘
```

- Markdown 是"事实陈述",CSV 是"索引摘要",两者同步发生。
- 人读双链、看 Graph;机器读注册表、跑查询与脱敏。
- 同一份事实,两种载体,AI 索引零歧义。

## 六、杠杆 × 复利(写进字段的思考)

每个目标/项目/资产携带:`leverage`、`asset_type`、`compounding`、`passive_income`。
让系统在每次记录时都追问:**"有没有更高杠杆的方式,做同一件事?"** 把时间从"线性消耗"变成"复利资产"。

## 七、隐私五分级

| 等级 | 含义 | 能否公开 |
|---|---|---|
| `public` | 可公开 | 是 |
| `internal` | 内部/脱敏后可分享 | 视情况 |
| `private` | 个人私域、家庭、健康 | 否 |
| `sensitive` | 客户/财务/合同 | 否,仅本地 |
| `secret` | 密钥/证件 | 永不入库 |

配合白名单导出 + fail-closed 默认:**未明确标记可公开的,一律不公开。**

## 八、健康度指标(系统的体检)

| 指标 | 健康阈值 |
|---|---|
| 孤立节点比例 | < 5% |
| 平均关联数 | ≥ 2 |
| 目标-项目连接率 | 100% |
| 复盘完成率(周) | ≥ 80% |

低于阈值,进入下一次复盘的警报清单。系统会告诉你它哪里"生病"了。
