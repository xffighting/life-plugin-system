# Skill · 新增内容入网 / Content Intake

> 一个让 AI Agent 把"任何一份散落的新内容"安全、规范地接入你人生图谱的技能。
> 它把[入网七步协议](../docs/architecture.md#四入网七步给人也给-agent-的协议)变成 Agent 可执行的步骤。

---

## 输入

- 一段原始内容:一条事实、一个想法、一段对话摘要、一份资料链接,或一个文件。
- 可选:你给的初步提示(它大概属于哪个领域/类型)。

## 输出

- 一张建好的卡片:含规范文件名、完整 frontmatter(`parent` + `related` 已填)、正文结构。
- 一条注册表更新建议(供你确认后写入)。
- 一份"我做了什么 / 还需你确认什么"的简短说明。

## 步骤(Agent 执行)

1. **判型** → 对照节点类型,判定它是 事实 / 知识资产 / 人物 / 实践 / 目标 / 项目 / 其他。
   - verify:类型唯一、不含模糊词(别把"练习"当"技能",别把"资料"当"知识资产")。
2. **命名** → 生成 `<前缀>-<语义化短名>`,文件名表达可读标题。
   - verify:同名不同类型必须拆分。
3. **建卡** → 选对应模板,填齐 frontmatter,尤其 `parent` 与 `related`(≥1)。
   - verify:没有空的必填字段;`privacy` 已设。
4. **连接** → 加 1 条上级边 + ≥1 条相关边;涉及目标/项目/人物时加对应关系。
   - verify:不产生孤立节点。
5. **脱敏检查** → 若内容可能对外,过 `automations/deid_gate.py --gate`。
   - verify:退出码 0。
6. **登记建议** → 给出注册表该新增/更新哪一行(交你确认,不擅自写真实数据)。

## 质量检查

- [ ] 类型判定唯一、无模糊词。
- [ ] frontmatter 必填齐全,`privacy` 合理。
- [ ] 至少 1 条 `parent` + 1 条 `related`,无孤立节点。
- [ ] 涉及隐私的内容已脱敏或标 `private/sensitive` 留本地。
- [ ] 留痕:在卡片变更日志记录"由 AI 建议生成,待人工确认"。

## 协同纪律

只读优先 · 建议先行 · 隐私拦截 · 痕迹留存。Agent 提出"新增/修改"建议,落库与否由你拍板。

## 示例(虚构)

> 输入:"今天和 `张示例` 聊了落地页转化,他建议先做小流量测试。"
> 输出:建议建 `EVENT-与张示例交流落地页转化` 人际事件卡,`related: [[PERSON-张示例]] [[PROJECT-上线一个落地页]]`,`privacy: private`,并提示更新人际事件注册表。
